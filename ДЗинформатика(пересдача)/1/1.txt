1. В основе стандрантной библиотеки лежат концепции
а) Переносимости (одинаковое поведение на всех системах)
б) Эффективности/ комп (баланс между компактностью кода и эффективностью реализации)
в) Связывание (выступает связующим звеном между другими библиотеками)
2. Система контроля версий используется при разработке проектов, над которыми работают несколько человек, для обеспечения своевременного доступа разработчкиков к новым версиям продукта и обеспечения CI/CD.
3. а) clone - клонирует существующий репозиторий
б) pull - "подтягивнаие" изменений из удаленного репозитория
в) commit - сохранение изменений
г) push - отправка изменений в удаленный репозиторий
д) merge - слияние веток разработки
4. Отдельные ветки следует создавать для разработки любых обновлений, чтобы не повредить рабочую версию продукта.
5. Основные элементы библиотеки chrono(библиотека для работы со временем и часами, библиотека интервалов и моментов времени)
а) интервал (duration) - n тактов заданной продолжительности 
б) момент времени (time point) - комбинация интервала и эпохи(начало отсчета)
в) часы (clock) - задают эпоху и продолжительность такта
